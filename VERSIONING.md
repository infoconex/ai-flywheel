# Versioning

This document explains how versions are handled for the Infoconex AI Flywheel Specification.

## Specification Versions

A specification version identifies the content and maturity of the Infoconex AI Flywheel Specification at a point in time.

Specification versions are assigned by Infoconex.

Examples:

- Infoconex AI Flywheel Specification 0.1
- Infoconex AI Flywheel Specification 0.1.1
- Infoconex AI Flywheel Specification 1.0

## Draft Versions

Versions below 1.0 are draft versions.

Draft versions may change as terminology, principles, lifecycle requirements, conformance guidance, examples, implementation experience, and research evolve.

A draft version should not be treated as permanently stable unless Infoconex says otherwise.

## Stable Versions

A 1.0 or later version may be identified by Infoconex as a stable specification version.

A stable version means the published specification is intended to be used as a reliable reference for implementation and conformance assessment.

Stability does not prevent future versions, corrections, clarifications, or revisions.

## Repository Releases

A repository release is not automatically a specification version.

A repository release may include documentation changes, site updates, examples, assets, research, or supporting materials.

A repository release identifies a specification version only when Infoconex explicitly says that it does.

## Website Deployments

A website deployment is not automatically a new specification version.

The website may be updated for formatting, navigation, images, search, styling, or other presentation changes without changing the specification version.

## Reference Implementations

A reference implementation, sample implementation, tool, script, or software release has its own versioning unless Infoconex explicitly states otherwise.

Implementation versions do not redefine the Infoconex AI Flywheel Specification.

## Conformance and Versions

A conformance claim must identify the specification version being used.

Recommended wording:

> This implementation is designed to conform to the Infoconex AI Flywheel Specification 0.1.1.

A conformance claim for one version does not automatically apply to a later version.

## Corrections and Clarifications

Infoconex may publish corrections, errata, clarifications, or explanatory guidance for a specification version.

A correction or clarification does not create a new specification version unless Infoconex identifies it as one.

## Deprecated Versions

Infoconex may deprecate, replace, or withdraw a specification version.

Deprecated versions may remain available for historical reference, but they should not be treated as current unless Infoconex says otherwise.

## Official Version Authority

Infoconex retains authority to assign, publish, revise, interpret, deprecate, and withdraw specification versions.
