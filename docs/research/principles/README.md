# AI Flywheel Principles

This section provides a detailed explanation of the eight principles on which the AI Flywheel is based.

The principles describe how an AI Flywheel should operate, how authority is governed, how work is distributed among different operating mechanisms, how execution produces learning, and how that learning compounds over time.

## Table of Contents

- [Principle 1: Autonomy Is Bounded by Human Authority](01-human-authority.md) — Defines delegated autonomy, Governance Policy, authority boundaries, uncertainty escalation, approval, and the role of human decisions as learning evidence.
- [Principle 2: AI Is the Operator](02-ai-as-operator.md) — Defines the shift from AI as a human assistant to AI as the operator of a persistent process and its capabilities.
- [Principle 3: Moving Determinism Boundary](03-moving-determinism-boundary.md) — Defines how responsibility is distributed among deterministic capability, procedural guidance, and AI reasoning and how that allocation can change over time.
- [Principle 4: SOP as Operational Control Plane](04-sop-control-plane.md) — Defines the SOP as a persistent machine-consumable contract governing execution and exceptions within human-defined governance.
- [Principle 5: Outcome Evidence](05-outcome-evidence.md) — Defines the requirement that execution produce evidence sufficient to evaluate actual outcomes.
- [Principle 6: Evolution Routing](06-evolution-routing.md) — Defines how observed weaknesses are classified and routed to the persistent destination best suited to improve them.
- [Principle 7: Persistent Operational Learning](07-persistent-learning.md) — Defines the requirement that useful learning survive the current execution in a reusable operational asset.
- [Principle 8: Compounding Reuse](08-compounding-reuse.md) — Defines how persistent improvements create cumulative operational leverage across repeated executions.

For the concise canonical statement of all eight principles, see the [AI Flywheel principles specification](../../specification/principles.md).
