# Stage 1: Execute

The AI performs the operational work using the current combination of procedural guidance, AI reasoning, and deterministic capability.

## Purpose

Execution is where the current operating system of the Flywheel is applied to real work.

The three operating mechanisms participate together:

- **Procedural guidance** defines how the process should be performed.
- **AI reasoning** interprets context, orchestrates actions, and handles ambiguity or judgment.
- **Deterministic capability** performs reliable, repeatable operations.

These are not separate lifecycle stages. A single execution may move between AI reasoning and deterministic tools many times while following the same procedure.

## Governance During Execution

Every action remains subject to the Governance Policy.

When execution reaches an authority boundary or an unresolved uncertainty boundary, the affected decision is escalated.

Where practical, unrelated authorized work should continue rather than stopping the entire process.

## Execution Produces the Raw Material for Learning

The Flywheel can only improve from what it actually does.

Execution therefore creates the events, outputs, errors, decisions, and state changes that the next stage observes.
